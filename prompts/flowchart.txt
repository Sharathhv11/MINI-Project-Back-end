- process:        Rectangle
- decision:       Diamond
- input_output:   Parallelogram
- loop:           Hexagon
- function_call:  Rectangle with double vertical borders
- declaration:    Rectangle with dashed border
- return:         Rounded rectangle with bold border

LAYOUT RULES:
1. Start node always at the top center.
2. Flow goes top to bottom for sequential steps.
3. Decision nodes branch left (false/no) and right (true/yes), or down (true) and right (false).
4. Loop back-edges connect from bottom of loop body back up to the loop node.
5. End node at the bottom center.
6. Minimum vertical spacing: 150px between nodes.
7. Minimum horizontal spacing: 250px for branches.

EDGE LABELS:
- Decision true branch: "Yes" or "True" or the condition value.
- Decision false branch: "No" or "False" or "else".
- Loop entry: "Loop" or the iteration condition.
- Loop exit: "Done" or "Exit loop".
- Normal flow: "" (empty string) or a brief description.

COMMON MISTAKES TO AVOID:
- Missing the start or end terminator node.
- Not connecting all branches back to the main flow.
- Forgetting loop back-edges.
- Using the wrong symbol type (e.g., process instead of decision for an if-statement).
- Not handling else branches or default cases.
- Orphaned nodes with no incoming or outgoing edges.
